# **🚀 Apply Now: Recruitment Platform UX/UI Case Study**


## **📖 Overview**

**Apply Now** is a conceptual, simple yet powerful recruitment platform (ATS) designed to allow HR departments to effortlessly start recruitment campaigns, track applications, evaluate applicants, and securely store candidate data for future reference.

This repository documents the end-to-end UX research, architectural planning, and high-fidelity UI design process created as part of the Product Designer assignment.

## **🎯 The Challenge**

The core objective was to design a platform that solves the common frustrations of modern recruiters—specifically "click fatigue," rigid data parsing, and poor historical talent archiving.

**Key Deliverables Required:**

1. User Flow & UX Research Synthesis  
2. High-Fidelity Desktop Designs for two core views:  
   * **Candidates List:** A highly scannable, filterable list of applicants.  
   * **Candidates Info Page:** A detailed workspace for evaluating an individual profile.

## **🧠 The Design Process**

To ensure the platform was built on actual human needs rather than assumptions, I followed a structured UX methodology:

### **1\. Research & Discovery**

* **Competitive Analysis:** Evaluated industry leaders (Workable, Greenhouse, Lever) to establish a baseline and identify opportunities to reduce cognitive load.  
* **Generative User Interviews (Simulated):** Created 3 core personas representing diverse hiring needs:  
  * *Sarah (High-Volume Recruiter):* Needs speed, scannability, and bulk actions.  
  * *David (Technical Recruiter):* Needs deep context, split-screen evaluation, and easy collaboration.  
  * *Elena (HR Director):* Needs data integrity and a searchable "Talent Vault" for future reference.

### **2\. Synthesis & Definition**

* **Empathy & Affinity Mapping:** Grouped pain points into actionable themes (e.g., eradicating click fatigue, frictionless bulk processing).  
* **User Stories:** Defined specific, feature-focused requirements based on persona goals.  
* **Evaluative UI Research:** Conducted a second round of inquiry specifically targeting layout expectations (e.g., persistent left-hand filters, sticky action headers).

### **3\. Information Architecture**

* **Sitemap:** Mapped the global navigation and drill-down paths from campaigns to individual candidate profiles.  
* **End-to-End User Flow:** Charted the user's journey from campaign creation, through high-volume screening, to final evaluation and talent archiving.

## **✨ Core Features & UI Solutions**

### **📍 View 1: The Candidates List**

Designed for high-volume screening and rapid decision-making.

* **Persistent Filters:** "Dealbreaker" filters (Location, Right-to-Work) are permanently pinned to the left sidebar for instant list manipulation.  
* **The "6-Second Rule" Grid:** Essential data (Current Title, Location, Match Score) is visible immediately without requiring expanding rows.  
* **Contextual Bulk Actions:** A sticky action bar appears dynamically only when multiple candidates are selected, allowing for frictionless bulk-rejects or stage advancements.

### **📍 View 2: The Candidates Info Page**

Designed to make candidate evaluation truly effortless.

* **The Split-Screen Paradigm:** Eradicates "click fatigue" by rendering the candidate's PDF resume natively on the left, while maintaining an active workspace on the right.  
* **Unified Evaluation Workspace:** Tabbed interface separating the Scorecard, Team Notes (with @tagging), and History.  
* **Sticky Dispositions:** \[Advance\] and \[Reject\] buttons remain pinned to the top right, ensuring recruiters can make decisions instantly without scrolling back to the top of a long resume.  
* **The Talent Vault integration:** Clear badging and timeline views if a candidate has applied to the company previously.
